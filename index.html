dnsxd

A DNS Service Discovery focused DNS server

dnsxd supports the protocols needed to do wide-area service discovery with Apple Bonjour:

  • DNS Update - to allow clients to create DNS records for services
  • DNS Update Leases - to allow records to expire if a client loses network connectivity
  • DNS Long Lived Queries (LLQ) - to allow clients to receive real-time query updates

All DNS Update messages must be authenticated with Transaction Signatures (TSIGs) and clients can optionally be restricted to only manipulating DNS records used to do DNS Service Discovery.

Limited DNSSEC support is available in the form of RSA SHA1 signatures and NSEC3 white-lies for authenticated non-existence.

Zone and related data is stored in CouchDB and can be replicated to other servers via CouchDB's built-in replication functionality. Other storage backends can be selected via a configuration option.

dnsxd is written in Erlang and released under the Apache License 2.0. Source code is available via git from GitHub. Archives will be available in the future. dnsxd is a young project and is yet to be proven in production.
